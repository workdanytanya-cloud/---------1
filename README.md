# Портфолио — Vibe Coder

Статический одностраничный сайт-портфолио: HTML, CSS, фоновая анимация Galaxy (WebGL через [OGL](https://github.com/oframe/ogl)).

## Структура

| Файл | Назначение |
|------|------------|
| `index.html` | Разметка и контент |
| `styles.css` | Оформление и адаптив |
| `galaxy.js` | Фон «звёздное небо», ES-модуль |
| `моё Фото.png` | Фото для hero и шапки (можно заменить своим, имя в коде — см. ниже) |

## Локальный просмотр

Из соображений безопасности браузеры ограничивают `import` из CDN при открытии файла как `file://`. Рекомендуется локальный HTTP-сервер.

**Вариант 1 — Python:**

```bash
cd Портфолио
python -m http.server 8080
```

Откройте в браузере: `http://localhost:8080`

**Вариант 2 — Node (npx):**

```bash
cd Портфолио
npx --yes serve .
```

## Публикация на GitHub

1. Создайте пустой репозиторий на GitHub (без README, если уже есть локальный коммит).
2. В папке проекта выполните:

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПО.git
git branch -M main
git push -u origin main
```

## GitHub Pages

1. Репозиторий → **Settings** → **Pages**.
2. **Source**: Deploy from a branch, ветка **main**, папка **/ (root)**.
3. Через минуту сайт будет доступен по адресу вида `https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПО/`.

Если вместо корня используете подпапку, в `index.html` при необходимости задайте `<base href="/ИМЯ_РЕПО/">` или относительные пути оставьте как есть — для корня репозитория они уже корректны.

## Своё фото

Файл в проекте: **`моё Фото.png`**. В `index.html` пути заданы в URL-кодировке; при переименовании обновите атрибуты `src` у аватарки в шапке и у большого фото в hero.
